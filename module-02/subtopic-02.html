Diffie-Hellman Key Exchange

Diffie-Hellman Key Exchange (DHKE) was introduced in 1976 by Whitfield Diffie and Martin Hellman, making it one of the earliest practical implementations of public-key cryptography. It solved a problem that had stumped cryptographers for decades: how can two parties who have never met privately establish a shared secret key, even if an eavesdropper can see their entire conversation?

The protocol doesn't encrypt messages directly. Instead, it provides a secure way for two parties to agree on a shared secret key, which they then use with a symmetric cipher (like AES) to encrypt actual communications. This makes Diffie-Hellman a key agreement protocol rather than an encryption algorithm.

How the Diffie-Hellman Exchange Works

Step 1 — Agree on public parameters

Alice and Bob publicly agree on a large prime number p and a generator g (a primitive root modulo p). These values can be known to everyone, including an eavesdropper.

Step 2 — Generate private secrets

Alice picks a secret integer a and computes A = ga mod p. Bob picks a secret integer b and computes B = gb mod p. Neither shares their private integer.

Step 3 — Exchange public values

Alice sends A to Bob; Bob sends B to Alice. This exchange happens over the public channel — an eavesdropper sees both A and B.

Step 4 — Compute the shared secret

Alice computes s = Ba mod p. Bob computes s = Ab mod p. Both arrive at the same value: gab mod p. This shared secret s was never transmitted.

Step 5 — Derive a symmetric key

The shared secret is fed into a key derivation function (KDF) to produce the actual encryption key used for the session.
